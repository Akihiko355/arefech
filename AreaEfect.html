✦ Area Effect GUI

Dokumentasi Referensi Fitur Tambahan untuk AtlantisCore Plugin (Paper 26.1.2)

Informasi Dasar

Command: /area effects <area_name>

Permission: atlantiscore.admin

API Target: Paper 26.1.2 (Adventure API & Legacy Component Serializer)

Config Default Refresh: Diambil dari area_defaults.effect_refresh_every (Default: 3s)

Aturan Teknis GUI

  • Membatalkan seluruh InventoryClickEvent (Item tidak bisa dipindah).
  • Format tulisan wajib menggunakan sistem Ampersand (&) melalui Adventure API. Dilarang keras memakai ChatColor lama.
  • Proses In-place Refresh GUI dilakukan dengan jeda Scheduler Bukkit agar tidak crash saat inventori dibuka di dalam event klik.

Alur Navigasi Player (Flow)

1. Jalankan Perintah: Admin mengetik command untuk membuka GUI 1 (Daftar Efek).

2. GUI 1 (Menu Utama - Maks 10 Slot Efek):

  • Klik slot kosong / tombol TAMBAH → Membuka GUI 2 (Mode ADD).
  • Klik slot efek yang sudah ada → Membuka GUI 2 (Mode EDIT).
  • Klik SIMPAN → Data masuk ke areas.yml. Klik TUTUP → Perubahan dibatalkan.

3. GUI 2 (Editor Efek):

  • Pilih jenis efek dari daftar halaman (diurutkan berdasarkan efek Beacon terlebih dahulu).
  • Atur Level via tombol +/- (Maksimum dibatasi sesuai tabel di bawah).
  • Atur Durasi via tombol kelipatan detik, atau aktifkan mode UNLIMITED (∞).
  • Simpan untuk kembali ke GUI 1 dengan status siap pakai, atau hapus efek (khusus Mode EDIT).

Struktur Penyimpanan Data (areas.yml)

Sistem menyimpan daftar efek pasif di dalam berkas areas.yml dengan format berikut:

passive_effects:
  - effect: SPEED
    amplifier: 0
    duration: 60
    unlimited: false
    refresh_every: 3
  - effect: HASTE
    amplifier: 1
    duration: -1
    unlimited: true
    refresh_every: 3
